## A) Brute-Force-Angriff auf ein Web-Login

### Hintergrund

Die Caesar-Chiffre zeigt das grundlegende Problem schwacher Schlüsselräume: Wenn es nur 25 mögliche Schlüssel gibt, probiert ein Angreifer sie einfach alle durch. Dasselbe Prinzip gilt für schwache Passwörter: Ein Angreifer sendet automatisiert Tausende von Login-Anfragen mit einer Passwortliste, bis er den richtigen Treffer landet. In dieser Aufgabe erleben Sie diesen Angriff live – gegen eine eigene, verwundbare Login-App auf Ihrer EC2-Instanz.

| HTTP | TCP | 80 | **My IP** |

```python
    sys.stdout.flush()

        break

elapsed = time.time() - start
print()
print(f"{'-'*40}")

if found:
    print(f"✓ Passwort gefunden: '{found}'")
    print(f"  Versuche: {i} | Zeit: {elapsed:.2f}s")
```

## B) AES-256 symmetrische Verschlüsselung

### Hintergrund

AES (Advanced Encryption Standard) wurde 2001 als Nachfolger von DES standardisiert. Mit einem 256-Bit-Schlüssel und dem GCM-Modus (Galois/Counter Mode) bietet AES-256-GCM sowohl Vertraulichkeit als auch Integrität der Daten – ein Angreifer kann den verschlüsselten Text nicht unbemerkt manipulieren.

### Aufgabe

Installieren Sie die Python-Cryptography-Bibliothek:

```bash
sudo apt update
sudo apt install python3-pip -y
sudo apt install python3-cryptography
```

Erstellen Sie das Verschlüsselungsskript:

```bash
nano ~/aes_demo.py
```

Fügen Sie folgenden Code ein:

```python
import os
from cryptography.hazmat.primitives.ciphers.aead import AESGCM

# Schlüssel generieren (256 Bit = 32 Bytes)
key = AESGCM.generate_key(bit_length=256)
aesgcm = AESGCM(key)

# Nonce generieren (96 Bit = 12 Bytes, einmalig pro Verschlüsselung)
nonce = os.urandom(12)

# Klartextnachricht
plaintext = b"Dies ist eine geheime Nachricht fuer M183!"

# Verschlüsseln
ciphertext = aesgcm.encrypt(nonce, plaintext, None)

print(f"Klartext:        {plaintext.decode()}")
print(f"Schlüssel (hex): {key.hex()}")
print(f"Nonce (hex):     {nonce.hex()}")
print(f"Ciphertext (hex): {ciphertext.hex()}")
print(f"Ciphertext-Länge: {len(ciphertext)} Bytes\n")

# Entschlüsseln
decrypted = aesgcm.decrypt(nonce, ciphertext, None)
print(f"Entschlüsselt:   {decrypted.decode()}")

# Manipulations-Test: Ciphertext verändern
tampered = bytearray(ciphertext)
tampered[0] ^= 0xFF  # erstes Byte kippen
try:
    aesgcm.decrypt(nonce, bytes(tampered), None)
except Exception as e:
    print(f"\nManipulations-Test: {e}")
    print("→ GCM-Modus hat die Manipulation erkannt!")
```

Führen Sie das Skript aus:

```bash
python3 ~/aes_demo.py
```

Beantworten Sie folgende Fragen schriftlich:

2. Was ist der Unterschied zwischen DES (56-Bit-Schlüssel) und AES-256 (256-Bit-Schlüssel) in Bezug auf Brute-Force-Resistenz?
3. Was demonstriert der Manipulations-Test am Ende des Skripts? Welchen Vorteil bietet GCM gegenüber einfachem AES-CBC?

## C) PKI-Zertifikatskette mit OpenSSL

### Hintergrund

S/MIME und HTTPS verwenden eine hierarchische Public Key Infrastructure (PKI). Die Vertrauenskette läuft von der **Root CA** (Vertrauensanker) über eine **Intermediate CA** bis zum **Server-Zertifikat**. Browser prüfen diese Kette beim Aufbau einer HTTPS-Verbindung.

In dieser Aufgabe erstellen Sie selbst eine solche Kette mit OpenSSL – genau wie eine echte Certificate Authority es tut.

```bash
[ Root CA ]            <- Der unfehlbare Boss (Fest im Browser eingebaut)
      │
      ▼
[ Intermediate CA ]    <- Der Stellvertreter (Erledigt die Alltagsarbeit)
      │
      ▼
[ Server-Zertifikat ]  <- Die Website (z. B. google.com)
```

### Aufgabe

Installieren Sie OpenSSL:

```bash
sudo apt install openssl -y
mkdir -p ~/pki/{ca,server} && cd ~/pki
```

**Schritt 1 – Root CA erstellen:**

```bash
# Private Key der Root CA
openssl genrsa -out ca/ca.key 4096

# Selbstsigniertes Root-Zertifikat (gültig 10 Jahre)
openssl req -new -x509 -days 3650 -key ca/ca.key -out ca/ca.crt \
  -subj "/C=CH/ST=Zuerich/O=TBZ-M183-CA/CN=M183 Root CA"
```

**Schritt 2 – Server-Zertifikat erstellen:**

```bash
# Private Key des Servers
openssl genrsa -out server/server.key 2048

# Certificate Signing Request (CSR) – Antrag an die CA
openssl req -new -key server/server.key -out server/server.csr \
  -subj "/C=CH/ST=Zuerich/O=TBZ-M183/CN=$(curl -s ifconfig.me)"
```

**Schritt 3 – CA signiert das Server-Zertifikat:**

```bash
# CA stellt das Server-Zertifikat aus (gültig 1 Jahr)
openssl x509 -req -days 365 \
  -in server/server.csr \
  -CA ca/ca.crt -CAkey ca/ca.key -CAcreateserial \
  -out server/server.crt

# Zertifikatskette erstellen (Server + CA)
cat server/server.crt ca/ca.crt > server/chain.crt
```

**Schritt 4 – Zertifikat prüfen:**

```bash
# Zertifikat-Details anzeigen
openssl x509 -in server/server.crt -text -noout | grep -A5 "Subject\|Issuer\|Validity\|Public Key"

# Zertifikatskette verifizieren
openssl verify -CAfile ca/ca.crt server/server.crt
```

Die letzte Ausgabe muss `server/server.crt: OK` zeigen.

3. Warum vertraut ein normaler Browser Ihrem selbst erstellten Zertifikat nicht, obwohl es technisch korrekt erstellt wurde?

## D) Nginx mit TLS konfigurieren

### Hintergrund

HTTPS ist ein **hybrides Verschlüsselungsverfahren**: Die asymmetrische Verschlüsselung (RSA/ECDH) wird für den sicheren Schlüsselaustausch verwendet, die eigentliche Datenübertragung erfolgt dann symmetrisch (AES). Genau das beschreibt der Modulinhalt unter «Hybride Verschlüsselungsverfahren».

### Schritt 1 – Port 443 freigeben

Navigieren Sie zu **EC2** → **Security Groups** → `m183-sg` → **Inbound rules** → **Edit inbound rules**. Fügen Sie folgende Regel hinzu:

| Type | Protocol | Port range | Source |
|------|----------|------------|--------|
| HTTPS | TCP | 443 | **My IP** |

### Schritt 2 – Nginx mit TLS starten

```bash
docker run -d \
  --name nginx-tls \
  -p 443:443 \
  -v ~/pki/server/chain.crt:/etc/nginx/ssl/server.crt:ro \
  -v ~/pki/server/server.key:/etc/nginx/ssl/server.key:ro \
  -v ~/pki/ca/ca.crt:/etc/nginx/ssl/ca.crt:ro \
  nginx:alpine
```

Erstellen Sie die Nginx-TLS-Konfiguration:

```bash
docker exec nginx-tls sh -c 'cat > /etc/nginx/conf.d/default.conf << '"'"'EOF'"'"'
server {
    listen 443 ssl;
    server_name _;

    ssl_certificate     /etc/nginx/ssl/server.crt;
    ssl_certificate_key /etc/nginx/ssl/server.key;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         HIGH:!aNULL:!MD5;

    location / {
        return 200 "<h1>M183 KN03 – TLS funktioniert!</h1><p>Hybride Verschlüsselung aktiv.</p>";
        add_header Content-Type text/html;
    }
}
EOF'

docker exec nginx-tls nginx -s reload
```

## E) HTTP vs. HTTPS – Traffic live mitlesen

### Hintergrund

Sie haben nun zwei Dienste gleichzeitig laufen:
- **Port 80** – die PHP-Login-App aus Aufgabe A (unverschlüsselt, HTTP)

### Schritt 2 – tcpdump: HTTP-Traffic mitlesen (Port 80)

Öffnen Sie **zwei SSH-Verbindungen** zu Ihrer Instanz gleichzeitig – Terminal 1 für tcpdump, Terminal 2 für den Login-Request.

**Terminal 1 – tcpdump starten:**


## F) Hash-Funktionen: MD5 cracken mit Python

### Hintergrund

Wie im Modulinhalt beschrieben, gilt MD5 als veraltet und unsicher für die Passwortspeicherung – nicht nur wegen Kollisionen, sondern vor allem weil MD5 **extrem schnell** berechnet werden kann. Ein moderner Rechner schafft Milliarden von MD5-Hashes pro Sekunde. Das bedeutet: Ein Angreifer, der eine gestohlene Passwort-Datenbank mit MD5-Hashes erbeutet, kann diese mit einem Wörterbuch-Angriff in Sekunden knacken.

In dieser Aufgabe spielen Sie genau diesen Angriff nach. Sie erhalten eine simulierte gestohlene Datenbank mit MD5-Hashes und cracken diese mit einem eigenen Python-Script und einer Passwortliste.

---

### Schritt 1 – Passwort-Datenbank und Cracker vorbereiten

Keine zusätzliche Installation nötig – Python3 ist bereits auf der Instanz vorhanden.

Erstellen Sie zuerst die simulierte gestohlene Hash-Datenbank. Sie enthält sechs Benutzer mit MD5-gehashten Passwörtern – genau so, wie man es in einem schlecht gesicherten System antrifft:

```bash

eve:4ece57a61323b52ccffdbef021956754
frank:e9f5bd2bae1c70770ff8c6e6cf2d7b76

```

Sie sehen nur Hashes – noch keine Passwörter. Das Format ist `benutzername:hash`, wie es typischerweise in einer Datenbank oder `/etc/shadow` vorkommt.

---

### Schritt 2 – Cracker-Script erstellen

Erstellen Sie das Cracker-Script:

```bash
nano ~/crack_md5.py
```

Fügen Sie folgenden Code ein:

```python
import hashlib
import time

HASHFILE  = "/root/hashes_md5.txt"
WORDLIST  = "/root/bruteforce-app/passwords.txt"

# Hashes einlesen
targets = {}
with open(HASHFILE) as f:
    for line in f:
        user, h = line.strip().split(":")
        targets[h] = user

# Wordlist einlesen
with open(WORDLIST) as f:
    passwords = [l.strip() for l in f if l.strip()]

print(f"Ziel-Hashes:  {len(targets)}")
print(f"Wörterbuch:   {len(passwords)} Einträge")
print(f"{'-'*45}")

found = {}
start = time.time()

for pw in passwords:
    h = hashlib.md5(pw.encode()).hexdigest()
    if h in targets:
        user = targets[h]
        found[user] = pw
        print(f"  ✓  {user:<10}  {h}  →  '{pw}'")

elapsed = time.time() - start

print(f"{'-'*45}")
print(f"Geknackt: {len(found)}/{len(targets)} | Zeit: {elapsed*1000:.1f} ms")
print(f"Hashes/Sekunde: {len(passwords)/elapsed:,.0f}")
```

```bash
password = b"sunshine"

# MD5: wie schnell?
start = time.time()
for _ in range(1000000):
    hashlib.md5(password).hexdigest()
md5_per_sec = 1000000 / (time.time() - start)

# scrypt (vergleichbar mit Argon2ID): wie schnell?
start = time.time()
for _ in range(10):
    kdf = Scrypt(salt=os.urandom(16), length=32, n=2**14, r=8, p=1)
    kdf.derive(password)
scrypt_per_sec = 10 / (time.time() - start)
print(f"scrypt: {scrypt_per_sec:>15,.1f} Hashes/Sekunde")

factor = md5_per_sec / scrypt_per_sec
print(f"\nMD5 ist {factor:,.0f}x schneller als scrypt.")
print(f"\nrockyou.txt (14 Mio. Eintraege) cracken:")
print(f"  Mit MD5:    {14000000 / md5_per_sec:.1f} Sekunden")
print(f"  Mit scrypt: {14000000 / scrypt_per_sec / 3600:.0f} Stunden")
```
